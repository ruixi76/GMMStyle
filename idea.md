# GMMStyle：像素级域自适应
我们首先将目标域数据以小批次的方式采样数据，然后通过EM算法和高斯混合模型对第一个批次的目标域像素进行聚类，得到k个高斯分布以及其比重，这k个高斯分布及其比例的权重和正好是目标域数据的分布。由于我现在正在研究的是域自适应，我希望模型可以学习到目标域数据的风格，所以我对源域数据与目标域数据进行风格混合。
首先，我们通过高斯混合模型得到了目标域图像的分布，让后对于第一个批次的源域数据进行风格混合，但是由于有k个高斯分布，我们需要得到源域数据中的像素与哪个高斯分布的概率最大，便记录为属于那个分布。让后利用如下公式进行风格迁移。如下为属于第一个高斯分布所进行风格混合的公式。
$$
I_{s1}'=\frac{I_s-\mu_s}{\sigma_s}\cdot\sigma_{t1}+\mu_{t1}
$$
其中关于μs和σs是源域一个小批次图像总的均值和方差。μt1和σt1是目标域生成的k个高斯分布的第一个均值和方差。
接下来，第二个批次我们对生成的k个高斯分布进行更新，第二个批次中的目标域图像的像素通过计算到达每个高斯分布的概率，来决定其属于哪个高斯分布，让后通过EM算法来进行更新，直至收敛。这时我们得到收敛的高斯分布和像素分布于那个高斯分布的列表。这时我们以第一个高斯分布进行举例，属于第一个高斯分布的像素进行计算均值μt2‘和方差σt2’，让后与第一个得到的进行混合，即如下公式：
$$
\mu_{t}^2=\mu_{t}^1\times a+\mu_{t}^{2'}\times(1-a) \\
\sigma_{t}^2=\sigma_{t}^1\times a+\sigma_{t}^{2'}\times(1-a)
$$
同上，我们对k个高斯分布进行更新。然后将源域进行风格混合的方式与第一次相同。我们本次得到的均值和方差的方法均不是通过特征提取所得到的，而是仅通过其原本图像的像素值进行计算得到。
## 关键模块
### 1. 基础模块：像素级 GMM 风格迁移 (Pixel-level GMM Style Transfer)
这是你最核心的想法：利用目标域图像拟合GMM，并将源域图像的像素对齐到目标域的高斯分布上。代码体现：style_transfer.py 中的公式应用，以及在 domain_adapter.py 中生成 styled_images ，并将其送入网络训练。
消融意义：验证“像素级对齐”本身是否比不作任何处理（Source-Only）更有效。
### 2. 损失模块：源域-风格域多样性/一致性约束 (Diversity / Consistency Loss)
在你的 train_epoch 损失计算中，你不仅使用了原始源域图像训练（loss_src），还引入了风格迁移后的图像一同训练（loss_style），并将二者结合。
代码体现：
`loss = loss_src + self.config.lambda_div * loss_style + ...`
消融意义：验证保留原始源域信息并将其与风格化图像共同作为约束，是否比单纯只用风格化图像训练更好。这证明了你的模型学习到了“内容不变性”，而不是仅仅去死记硬背目标域的颜色。
### 3. 监督模块：目标域高置信度伪标签 (Target Pseudo-labeling)
你在训练过程中，对目标域图像的前向传播结果进行了置信度过滤（>0.95），并计算了伪标签交叉熵损失。
代码体现：
`mask = max_probs.ge(self.conf_threshold).float()`，以及随后的 
`loss_tgt = (nn.CrossEntropyLoss()(logits_tgt, tgt_preds) * mask).mean()`。
消融意义：验证在像素级对齐的基础上，引入目标域的自监督信号是否能进一步拉近特征空间的高维语义距离。
### 4. 动态模块：GMM 增量更新机制 (Incremental GMM Update)
你并没有只在第一轮就把 GMM 固定死，而是每隔一定的 batch（config.gmm_update_freq），利用 EMA（指数移动平均，引入 momentum 或 $\alpha$）来平滑更新 GMM 的均值和方差。
代码体现：
`self.target_gmm.update_incremental(target_pixels, alpha=self.config.momentum)`。
消融意义：验证“动态感知”目标域分布变化，是否比“静态固定”初始化分布（Static GMM）更加鲁棒和精准。
### 5. 超参模块：软风格混合因子 (Soft Style Blending $\alpha$)
在进行风格迁移时，你并没有100%替换为目标域统计量，而是保留了一部分源域的原始像素值。
代码体现：
`styled_pixels = (normalized * sigma_t + mu_t) * alpha + source_pixels * (1 - alpha)`。
消融意义：通过调节 $\alpha$（如 0.0, 0.5, 1.0），证明适度的风格混合能防止语义信息的过度丢失（即“洗白”或“染黑”现象）。
